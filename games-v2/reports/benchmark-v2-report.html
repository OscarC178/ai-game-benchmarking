Key Findings

1. GPT-5 Mini was never broken — it was misconfigured.
The entire "Mini failure" narrative (24 DNFs, worst builder) was caused by a wrong model ID routing to an exhausted OpenRouter key. Once fixed, Mini completed 50/50 builds with 0 DNFs and scored 9.56 avg — mid-tier, not catastrophic.
2. GPT-5.4 flagship is the most reliable builder.
50/50 complete, 0 DNF, 9.92 avg. The only builder with zero catastrophic builds across all planners.
3. Sonnet 4.6 is the most spec-sensitive builder.
3 DNFs (all in r1-opus), 5 catastrophic builds with 100+ errors. Sonnet fails hard without good specs, but performs adequately with structured specs from any non-Opus planner.
4. Gemini 3.1 Pro specs produced the highest average scores.
r3-gemini-pro: 9.93 avg with 0 DNFs. Slightly ahead of GLM-5 (9.89) and GPT-5.4 (9.82).
5. Specs matter — but not as much as builder choice.
r5-control (no specs) still scored 9.70 avg. The gap between best planner (9.93) and no planner (9.70) is just 0.23 points. Builder choice creates a 0.58-point spread (9.92 vs 9.34).

Methodology Issues

Issue 1 — GPT-5 Mini Model ID (RESOLVED)
Agent bench-gpt54mini was configured with model ID openai/gpt-5.4-mini (wrong) instead of openai/gpt-5-mini (correct). This routed all Mini API calls to OpenRouter, which exhausted its credit limit after r1. Runs r2-r5 all returned 403 errors, producing 40 false DNFs. Fixed April 3, 2026 — all 40 builds re-run via palebluedot, all succeeded.
Issue 2 — Proxy Scoring
Score = max(0, 10 - JS_error_count). This is bimodal: 77% of builds score 10.0, 5 builds score 0.0. The scale doesn't discriminate between "nearly perfect" and "good" builds. Static QA was planned but never executed — the proxy is unvalidated.
Issue 3 — r1-opus is the only run with DNFs
All 5 DNFs and the 1 missing build come from r1-opus. This may indicate an issue with the r1 run environment rather than Opus specs specifically.
